## What was the problem you were solving in the projects for this course?

For these projects, I developed a course advising system for ABC University (ABCU). The goal was to create a program that could load course 
information from a file, store the information efficiently, display a complete list of courses in alphanumeric order, and allow users to search 
for a specific course and view its prerequisites. The project required evaluating multiple data structures to determine which would best meet the 
needs of the advising department while maintaining efficient performance.

## How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by first analyzing the program requirements and then evaluating how different data structures would 
affect performance. Throughout the course, we worked with vectors, hash tables, and binary search trees. Understanding data structures 
was important because each structure offers different advantages for storing, retrieving, and organizing data. By comparing their runtime 
performance and memory usage, I was able to determine which structure best supported the program's requirements. This process helped me 
understand that selecting the appropriate data structure can significantly improve both efficiency and functionality.

## How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges was understanding how to analyze the performance of different data structures and determine which one 
was most appropriate for a given problem. I overcame this by breaking larger problems into smaller steps, reviewing course materials, 
and testing different approaches through pseudocode before implementation. Another challenge was ensuring that course prerequisites were 
validated correctly and that the data was loaded accurately from the input file. Careful planning and repeated review of the project 
requirements helped me work through these issues.

## How has your work on this project expanded your approach to designing software and developing programs?

This project reinforced the importance of planning before coding for me. Creating pseudocode and analyzing different data structures allowed me to 
think critically about program design before implementation. Rather than immediately writing code, I learned to evaluate multiple solutions and 
consider factors such as efficiency, scalability, and maintainability. This approach has improved my problem-solving skills and helped me become 
more intentional in my software design decisions.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project reinforced the value of writing organized and modular code. By separating functionality into distinct operations such as loading data, 
searching, sorting, and displaying results, I gained a better understanding of how to create programs that are easier to read, maintain, and modify. 
I also learned the importance of meaningful variable names, clear program structure, and selecting data structures that support future growth. 
As a result, I now focus more on creating adaptable solutions that can be expanded or updated without requiring major changes to the overall program.
